# SP500 系統架構設計（v2，2026-07）

## 核心定位

SP500 = 個人信任連結庫，目標 500+ 條，涵蓋所有興趣領域。
不是新聞閱讀器，是**情報原料庫**。庫本身是活的，會隨行為數據漂移。

## 三層架構

| 層 | 職責 | 輸入 | 輸出 |
|----|------|------|------|
| Layer 1 | 已認證來源池 | 人工審定 + Layer 3 晉升 | sources.md |
| Layer 2 | 自動收割日常內容 | sources.md（有效 RSS） | 每日 inbox |
| Layer 3 | 候選驗質 + 晉升入庫 | candidates.md | sources.md 新增條目 |

## Layer 2 執行邏輯

收割優先順序依 F（更新頻率）× S（可抓取性）決定：
- F≥8, S≥8 → 每 4 小時 RSS 自動收割
- F≥8, S<6 → Layer 3 定期 web_fetch（非 RSS）
- F<5 → 週批次，不佔每日額度

輸出格式：
```
日期 | 來源縮寫 | 標題 | 評分(O,F,S,C,A) | URL | status:inbox
```

## Layer 3 執行邏輯

1. 從 candidates.md 取待審來源
2. web_fetch 最新 3 篇文章
3. AI 判斷：原創性、更新頻率、語言品質
4. 通過 → 附驗證日期寫入 sources.md
5. 不通過 → 標記原因留在 candidates.md

RSS 為 none 但品質高的來源 → 備註標記 `fetch:periodic`，進 Layer 3 週期爬取路徑。

## 興趣領域覆蓋目標

**已涵蓋：** 科技 / 財經 / 電影 / 人文
**待補齊：** 心理 / 性別議題 / 創業 / 經濟學 / 人類學 / 日本漫畫 / 主機遊戲 / 串流上片 / 育兒 / 居家整理 / 寶可夢卡牌 / 兩性戀愛

## 興趣圖譜演化機制

靜態層：sources.md 評分（人工宣告）
動態層：晨報愛心紀錄（行為數據）

演化規則：
- 來源愛心率高 → 評分 A（權威）維度 +1，版面佔比增加
- 來源愛心率連續 30 天 < 5% → 觸發降權，標記 pending-review
- 愛心文章落地 Vault → 自動 intake 該來源進候選池（若不在庫中）

最終效果：庫的內容比重隨你的實際閱讀行為漂移，不需要人工重新盤點。
