## G1: 摘要結果未落地 Vault 就結案（⚠️ 通用坑）

**錯誤**：摘要跑完放在 session 記憶中就回覆使用者，沒有寫入 Obsidian。
**正確**：每次摘要完成後必須存到 `00_Inbox/` 或指定路徑，並確認檔案存在（ls 驗證）。session 結束後 context 會消失。
**觸發情境**：跑完 yt-summary.sh 後直接 Reply，沒有寫入 Vault。

---

## G2: GEMINI_API_KEY 429 時忘記切換到 Gemini CLI OAuth

**錯誤**：Gemini API key 觸及 429 rate limit 時繼續重試，或直接報錯回給使用者。
**正確**：系統已設定 Gemini CLI OAuth 作為備援（走 Google 訂閱制，不受 API key 限制）。429 時切換使用 `gemini -p "..."` 指令（pty=true）。
**觸發情境**：大量批次摘要或 API 額度吃緊時段。

---

## G3: 長影片字幕截斷導致摘要不完整

**錯誤**：假設字幕全部取到，直接摘要，結果只覆蓋影片前半段。
**正確**：取得字幕後先確認 token 數，超過 LLM context 限制需分段摘要再合併。回覆時說明是否為完整摘要。
**觸發情境**：2 小時以上的長講座、podcast 影片。
